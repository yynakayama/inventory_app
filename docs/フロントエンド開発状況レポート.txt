## ✅ 完成済み・進行中項目

### Phase 1: 認証・基盤（完了）
- JWT認証、権限制御（RouteGuard, PermissionGuard）、基本レイアウト、UIコンポーネント（Button, Modal等）の実装が完了。

### Phase 2: 主要機能実装（80%完了）

#### 1. ダッシュボード機能 (完了)
- **実装済み**:
  - KPIカード（不足部品、入荷予定など）による状況の可視化。
  - 緊急アラート（緊急不足、遅延入荷）の表示。
  - 各種サマリー（不足部品、入荷予定、仕入先、棚卸）の詳細表示。
  - バックエンドAPI (`/api/reports/dashboard`) との連携。

#### 2. 在庫管理機能 (完了)
- **実装済み**:
  - 在庫一覧の検索、フィルタ（生産不足、安全在庫割れ）、ページネーション。
  - 部品詳細モーダルでの詳細情報、入出庫履歴の確認。
  - 在庫調整（入庫・出庫）機能。
  - 棚卸機能（実地数量の入力、差異の記録）。
  - 部品コード選択コンポーネント (`PartCodeSelector`) の導入。

#### 3. 調達管理機能 (完了)
- **実装済み**:
  - 予定入荷一覧の検索、フィルタ。
  - 新規発注、納期回答、統合入荷処理、発注キャンセルの一連のワークフロー。
  - 「不足部品一覧」から直接発注する機能。
  - ステータスに応じたUI制御と色分け。

#### 4. 生産計画機能 (完了)
- **実装済み**:
  - 生産計画一覧の検索、フィルタ。
  - 新規生産計画の作成。
  - 所要量計算と不足部品の表示。
  - 生産開始（部材消費）、生産完了のステータス更新。
  - 部材不足がある計画の警告表示。

## 🎯 次フェーズの作業予定

### Phase 3: マスタ管理（最終フェーズ）
1. **部品マスタ管理画面**
   - 部品の一覧、登録、編集、削除機能。
2. **製品マスタ・BOM管理画面**
   - 製品とそれに紐づく部品構成（BOM）の管理機能。
3. **ユーザー管理画面** (管理者向け)
   - ユーザーの権限変更や追加・削除機能。

### Phase 4: テスト・最適化
1. **単体・結合テストの実装**
   - Jest, React Testing Libraryを用いたテストコードの作成。
2. **E2Eテストの実装**
   - Playwright等を用いた主要フローの自動テスト。
3. **パフォーマンス最適化**
   - 大量データ表示時の仮想化、バンドルサイズの削減など。
